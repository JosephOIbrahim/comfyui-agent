### ▸ Q1 — LIVRPS tick API contract (from first principles)

**Required derivation.**
*   **Read-only:** Distributed Systems (DS) frame. Observers of a logical clock cannot mutate it; otherwise, writers could poison the causality of the system. Enforced at the type-system level via language-native getter semantics (a Python `@property` with no implemented `__set__` method).
*   **Return type:** `uint64` (Python `int`). DS frame. A 64-bit integer provides 18 quintillion IDs. At an agent loop speed of 1,000 steps per second, it wraps in 584 million years. A structured `(epoch, tick)` tuple is rejected because R4 established strict WAL partitioning by session (`wal_{session_uuid}.jsonl`); the session UUID inherently acts as the epoch boundary. `uint128` introduces unnecessary serialization overhead.
*   **Strict monotonicity:** $T_{n+1} > T_n$. DS frame. The sequence must strictly increase across state mutations. **Failure semantics:** If the bridge observes a non-monotonic sequence during session-end consolidation (e.g., tick 45 followed by tick 42), it must raise a `CausalInversionError` and trigger the Last-Valid-Tick Truncation. Non-monotonic logs permanently poison the Control Theory loop.
*   **Idempotency:** Control Theory (CT) frame. Two reads within the *same* agent cognitive step **must return the exact same value**. If Brain JSONL, Cognitive JSONL, and USD Prims all fire asynchronously during Step 5, they must all be stamped with `5`. If reads auto-increment or rely on write-time wall-clocks, concurrent writes shatter into disjoint causal frames. Idempotency guarantees the `fsync` barrier cleanly batches exactly one cognitive step, which is mathematically required for reproducible Held-Out Replay artifacts.
*   **Bound-state coupling invariant:** *The sequence increments exclusively upon an irreversible cognitive state transition (intent generation), completely decoupled from bounded-queue evictions or ring-buffer modulo math.* It is an absolute cumulative odometer, not an array index.
*   **API name:** `causal_sequence_id`. "Tick" implies a temporal wall-clock relationship. "Sequence" maps to logical ordering. "Causal" enforces its Lamport-clock purpose: tying parallel disjoint writes back to the singular cognitive intent that spawned them.

**Fallback derivation.**
If the existing LIVRPS monotonic index is structurally bound to a recycled queue (and adding an independent odometer violates the immutability rule), we must use an external primitive.
*   *External Hybrid Logical Clock (HLC):* Overkill synchronization overhead for a single-node IPC setup.
*   *Bridge-maintained proxy (mtime):* Violates causal ordering (the bridge observes the write *after* the async delay, stamping it with the completion time rather than the intent time).
*   *Loop-Scoped Monotonic Snapshot:* `time.monotonic_ns()` sampled **exactly once** at the top of the LIVRPS `while` loop iteration, and passed by-value to all egress payloads.
    *   *Preserves:* Strict monotonicity, exact idempotency (all async writers in that iteration share the exact nanosecond integer), and fsync grouping.
    *   *Loses:* 1:1 scalar mapping to the internal LIVRPS UI visual states.

**Verdict.**
**Native LIVRPS reuse** (`causal_sequence_id`) is the default.
*Assumption Joe must confirm:* The LIVRPS state machine inherently maintains a `total_transitions` integer that operates independently of its ring-buffer `maxlen` eviction. If this assumption is false, the verdict strictly defaults to the **Loop-Scoped Monotonic Snapshot** fallback to avoid forbidden LIVRPS surgery.

**Counter-argument.**
The strongest case for *no logical clock* (wall-clock at write-time only) is zero-touch integration. *Rejection:* We lose the exact semantic grouping of "this image was generated by this specific prompt iteration." At v1.1 scale, timestamp proximity is an acceptable heuristic, but heuristics fail under VRAM OOM lag spikes. Explicit logical causality is non-negotiable.

### ▸ Q2 — Compression mechanism for Axiom 7 (from first principles)

**Required derivation per candidate.**

1.  **Terminal state extraction:**
    *   *Compression ratio:* ~100:1.
    *   *Preserves/Loses:* Preserves final accepted outcome. Loses all thrashing, errors, and reasoning trajectory.
    *   *Determinism & Cost:* 100% deterministic, zero cost.
    *   *Phase 2.5 compatibility:* **FATAL.** `signal_attention` explicitly requires iteration history to retroactively map which choices led to the outcome. Terminal state extraction destroys the failure topologies needed for negative weighting.
2.  **LLM summarization:**
    *   *Compression ratio:* ~50:1.
    *   *Preserves/Loses:* Preserves semantic narrative. Loses deterministic JSON payloads and literal vector geometry.
    *   *Determinism & Cost:* Non-deterministic (stochastic drift). High latency (2-10s) at session-end risks fsync timeouts.
    *   *Phase 2.5 compatibility:* **FATAL.** You cannot run an algorithmic optimizer prior over fuzzy natural language; it requires structured nodes.
3.  **Intent-topology deduplication:**
    *   *Compression ratio:* ~5:1 to 15:1.
    *   *Preserves/Loses:* Preserves the unique sequence of cognitive shifts (A → B → C) and discrete failure states. Loses high-entropy, zero-delta micro-thrashing (A → A → A).
    *   *Determinism & Cost:* 100% deterministic (string/hash matching), <50ms CPU time.
    *   *Phase 2.5 compatibility:* **EXCELLENT.** It hands Phase 2.5 exactly what it needs: a clean topological graph of distinct choices to attach retroactive confidence weights to.

**Verdict.**
**Intent-topology deduplication.** It maximizes the Information Theory (IT) frame by mathematically filtering high-entropy noise without dropping structural signal, protecting Moneta's retrieval channel capacity. It satisfies the Distributed Systems (DS) frame by executing synchronously and locally, averting network-timeout failure modes during fragile session teardowns.

**Counter-argument.**
The strongest case is for *no compression at all in v1.1*. Defer Axiom 7 to Phase 2.5 entirely to prevent premature data loss. *Rejection:* Uncompressed WALs contain 90% wait-state and retry loops. Dumping this directly into Moneta mathematically guarantees that associative retrieval will pull the agent toward past failure modes rather than past solutions.

The compression upgrade path is perfectly **orthogonal** to the Phase 2.5 boundary. In v1.1, intent deduplication is a *structuring* algorithm—it prunes the graph. The Phase 2.5 `signal_attention` work is a *weighting* algorithm. In Phase 2.5, the pipeline does not replace deduplication; it simply maps the vision evaluator's scores onto the deduplicated topological nodes v1.1 already produces.
